springboot集成mybatis后，开始事务管理

  1. 启用全局事务管理：在springboot的启动类上面，添加@**EnableTransactionManagement(proxyTargetClass = true)**注解
  2. 在service层中，在要添加的方法上面添加@Transactional注解，代表这个方法里面的操作是在同一个事务里面的，只要有一个事务有异常，就会回滚。

有了外键应该怎么删除数据

mybatis中parameterType中怎么传入多个不同类型的参数

不需要使用parameterType参数

  1. 使用下标

    public List<XXXBean> getXXXBeanList(String xxId, String xxCode);  
    <select id="getXXXBeanList" resultType="XXBean">

　　select t.* from tableName where id = #{0} and name = #{1}  

</select>

    由于是多参数那么就不能使用parameterType， 改用#｛index｝是第几个就用第几个的索引，索引从0开始

  2. 注解

    public List<XXXBean> getXXXBeanList(@Param("id")String id, @Param("code")String code);  
    <select id="getXXXBeanList" resultType="XXBean">

　　select t.* from tableName where id = #{id} and name = #{code}  

</select>

    由于是多参数那么就不能使用parameterType， 这里用@Param来指定哪一个

  3. Map封装

    public List<XXXBean> getXXXBeanList(HashMap map);  
    <select id="getXXXBeanList" parameterType="hashmap" resultType="XXBean">

　　select 字段... from XXX where id=#{xxId} code = #{xxCode}  

</select>

    其中hashmap是mybatis自己配置好的直接使用就行。map中key的名字是那个就在#{}使用那个，map如何封装就不用了我说了吧。

  4. List封装

    public List<XXXBean> getXXXBeanList(List<String> list);  
    <select id="getXXXBeanList" resultType="XXBean">
　　select 字段... from XXX where id in
　　<foreach item="item" index="index" collection="list" open="(" separator="," close=")">  
　　　　#{item}  
　　</foreach>  
</select>

    总结

    传递list和map在资源消耗上肯定远大于方法一和方法二，但是有一些特殊的情形需要传递list，比如你需要传递一个id集合并批量对id进行sql操作然后再返回等等。所以都需要了解。

Mybatis中传递参数时，会加上单引号吗

使用**#{参数}**传入会加上单引号，sql语句解析是会加上‘ ’

比如 select * from table where name = #{name} ,传入的name为小李，那么最后打印出来的就是

select * from table where name = ‘小李’，就是会当成字符串来解析

但是${}并不会加上单引号

select 时间 from 表名 group by 时间

在数据库中按照准确的时间应该怎么查

使用DATE_FORMAT()函数将date类型的转换为字符串

DATE_FORMAT(date,format)用法：将时间date转换为format格式的字符串

format是自己定义的格式，如DATE_FORMAT(date,'%Y-%m-%d')，就是将date转换为格式为年-月-日的字符串

select * from 表名 where DATE_FORMAT(date,'%Y-%m-%d')=xxx

@MapperScan注解的使用

在学习了spring之后，不在需要给dao层建立实体类，只需要写出dao的接口就行了，但是需要我们使用@Mapper注解，在dao接口类上添加@Mapper注解，在编译之后就会生成接口实现类，不在需要我们人为的去写一个实现类。

但是在接口类很多的情况下，就需要我们给很多接口类都添加@Mapper注解，比较麻烦。所以就出现了@MapperScan

@MapperScan注解用于指定dao接口类所在的包，该包下的每一个接口在编译之后都会生成一个实现类，不用再在每一个接口类上添加@Mapper注解。

@MapperScan注解在springboot启动类上添加

@MapperScan("com.hutdsj.blog.mapper")   //指定接口类所在的包，com.hutdsj.blog.mapper下的所有接口都会在编译后生成实现类@SpringBootApplication  //指定这是一个springboot程序public class BlogApplication {    public static void main(String[] args) {        SpringApplication.run(BlogApplication.class, args);    }}

@MapperScan注解指定多个包

@MapperScan({"com.kfit.demo","com.kfit.user"})    //使用逗号隔开

使用@Autowried 注解后注入为null

当你的注入对象已经注册到容器中（即已经添加了@Compoent，@Controller….等注解），但是仍然注入为null时，需要检查获取对象的方式是否为new,new出来的对象使用的是有参，无参构造，不走spring的IOC容器，直接创建出来的。而@Autowried注解是根据type自动配置，要求对象是在IOC容器里面的。

@GetMapping("/verifycode")public void getVerify(HttpServletRequest request, HttpServletResponse response) {    try {        response.setContentType("image/jpeg");//设置相应类型,告诉浏览器输出的内容为图片        response.setHeader("Pragma", "No-cache");//设置响应头信息，告诉浏览器不要缓存此内容        response.setHeader("Cache-Control", "no-cache");        response.setDateHeader("Expire", 0);        VerifyCode verifyCode = new VerifyCode();        verifyCode.getRandcode(request, response);//输出验证码图片方法    } catch (Exception e) {       e.printStackTrace();    }}
public class VerifyCode {    @Autowired    RedisUtil redisUtil;    ......    public void getRandcode(HttpServletRequest request, HttpServletResponse response) {     	.......        redisUtil.set(RANDOMCODEKEY,randomString);      	......    }}

如上面这两段代码，因为verifyCode这个对象是被new出来的，即使在VerifyCode这个类中使用 @Autowired标注了

RedisUtil,但是仍然获取不到RedisUtil的值，即没有创建出这个对象。

解决方法是在第一段代码中，verifyCode这个对象不要是new出来的，使用 @Autowired标注实体类 

@AutowiredVerifyCode verifyCode;

简单的来说：**@Autowired注入Spring Bean，则当前类必须也是Spring Bean才能调用它，不能用new xxx()来获得对象，这种方式获得的对象无法调用@Autowired注入的Bean。**

使用这个注解给实体类注入信息，需要配置文件中的属性名和实体类中的属性名一致，同时实体类中的属性还需要有set,get方法。

在拦截器中使用@AutoWired注解注入为null

因为拦截器的加载是在springcontext之前，所以在拦截器中注入为null。

因此我们需要在配置类中使用@Bean注解提前去加载

修改前的配置类

@Configurationpublic class WebConfig implements WebMvcConfigurer {        //注册拦截器    @Override    public void addInterceptors(InterceptorRegistry registry) {        //添加具体的拦截器，同时设置拦截的地址。        //JwtInterceptor为我们拦截器实体类        registry.addInterceptor(new JwtInterceptor()).addPathPatterns("/user/h");    }}

修改之后：

@Configurationpublic class WebConfig implements WebMvcConfigurer {	//使用@Bean注解提前加载JwtInterceptor这个类    @Bean    public JwtInterceptor jwtInterceptor(){        return new JwtInterceptor();    }    //注册拦截器    @Override    public void addInterceptors(InterceptorRegistry registry) {        //添加具体的拦截器，同时设置拦截的地址。        registry.addInterceptor(jwtInterceptor()).addPathPatterns("/user/h");    }	//注意：我们这里使用@Bean将jwtInterceptor 注册到IOC容器，然后下面addInterceptor中的参数不再是new JwtInterceptor(),而是调用上面的jwtInterceptor()方法}

Disconnected from the target VM, address: ‘127.0.0.1:3231’, transport: ‘socket’

在项目中报了这个错，在网上百度了很久，都不是我错误的原因。

我找了很久，发现是将一个类两次注册到IOC容器中的原因。

保存到数据库的时间和设置的时间相差8个小时

在此之前，我写jdbc的url都是这样写的url:jdbc:mysql://localhost:3306/xxx?serverTimezone=utc

我们将时区设置为utc（utc不是时区，这样说只是为了理解），但是utc是协调世界时，中国在东八区，和这个时间相比快了8个小时，因此我们需要将时区设置为中国的时区

所以以后我们的url可以这么写: url: jdbc:mysql://localhost:3306/hutbbs?serverTimezone=Asia/Shanghai

从前端一次接受多个参数应该怎么做

有时候前端可能一次性将填写的多个同一类型的数据提交给后端，这个时候我们需要使用list接收数据，在使用list接收数据的时候，还需要使用@RequestBody这个注解。

将多个对象同时传入数据库，可以使用foreach来遍历。

<insert id="insert" parameterType="list">    insert into ApplyData (id, datatype, Idcard,      gettime, `name`, `rank`,      `group`, `code`, checkstatus,      checktime, checkpeople, IntoTime      )      values    <foreach collection="list" item="item" index="index" separator=",">       (#{item.id,jdbcType=INTEGER}, #{item.datatype,jdbcType=INTEGER}, #{item.idcard,jdbcType=NVARCHAR},      #{item.gettime,jdbcType=NVARCHAR}, #{item.name,jdbcType=VARCHAR}, #{item.rank,jdbcType=NVARCHAR},      #{item.group,jdbcType=VARCHAR}, #{item.code,jdbcType=NVARCHAR}, #{item.checkstatus,jdbcType=NVARCHAR},      #{item.checktime,jdbcType=TIMESTAMP}, #{item.checkpeople,jdbcType=NVARCHAR}, #{item.intotime,jdbcType=TIMESTAMP}      )    </foreach></insert>

其中collection是容器的类型，item是容器中的每一个对象的名字，separator是分隔符。

注意只要将数据放在foreach中即可，我在增加的时候，不小心将values这个字段一同放在foreach中，结果插入第二条数据死活插入不进去，看了别人的foreach的用法才知道values应该放在外面

使用foreach查询数据

碰到一个需求，传过来一个string集合，然后根据string集合中的每一个元素去数据库查询相对应的数据。一看到参数是个集合，我就想到应该使用foreach来进行遍历，的确也是使用这个。

我以为查询和上面的插入是一样的，因此直接套用了上面插入的语法，但是发现死活都插不进去。后面打印了sql语句才知道问题。

<select id="getDescribe" resultType="optioncode">        select CodeNo,`Describe`,typename,count from OptionCode        where  CodeNo =        <foreach collection="list" item="item" index="index" separator=",">                  #{item}         </foreach>    </select>

这是最初的写法，可以发现foreach和上面的插入是一样的。

[A01, A02, A03]Creating a new SqlSessionSqlSession [org.apache.ibatis.session.defaults.DefaultSqlSession@3eed9c44] was not registered for synchronization because synchronization is not activeJDBC Connection [com.alibaba.druid.proxy.jdbc.ConnectionProxyImpl@418882cf] will not be managed by Spring==>  Preparing: select CodeNo,`Describe`,typename,count from OptionCode where CodeNo = ? , ? , ?==> Parameters: A01(String), A02(String), A03(String)

这是打印出来的sql语句，select CodeNo,Describe,typename,count from OptionCode where CodeNo = ? , ? , ? 我们发现语句是 xxx= xx,xx,xx 很明显，等号并没有这个作用，等号只能附一个值，也就是说只能是xxx=xx这种，所以我之前只传一个值的时候是可以查询到数据的。当后面的具有多个参数的时候，我们就想到应该使用in这个关键字，in后面可以是一个集合。

但是in的格式是 xxx in (xx,xx,xx),所以我们就需要给传过来的参数添加一对括号，因此，需要在foreach中进行修改。修改的代码如下

select CodeNo,`Describe`,typename,count from OptionCode        where  CodeNo in        <foreach collection="list" item="item" index="index" separator="," open="(" close=")">                  #{item}         </foreach>

这是打印出来的sql语句

[A01, A02, A03]Creating a new SqlSessionSqlSession [org.apache.ibatis.session.defaults.DefaultSqlSession@6bd28c3b] was not registered for synchronization because synchronization is not activeJDBC Connection [com.alibaba.druid.proxy.jdbc.ConnectionProxyImpl@6d0e99e0] will not be managed by Spring==>  Preparing: select CodeNo,`Describe`,typename,count from OptionCode where CodeNo in ( ? , ? , ? )==> Parameters: A01(String), A02(String), A03(String)

可以很明显的看见这里已经变成了in ,select CodeNo,Describe,typename,count from OptionCode where CodeNo in ( ? , ? , ? ) 这样查询是可以查询到的。
