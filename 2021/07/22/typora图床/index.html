typore图床

发表于 2021-07-22

写作背景

我都是使用typora来记笔记，在写笔记的时候，难免会需要贴一些图片，而这些图片都是保存到本地的，如果你需要把这个文章分享给别人的话，就必须将图片也传给他，并且他还需要将图片保存到和你一样的位置，非常麻烦。在将文章部署到github上的博客时，图片的问题也是很麻烦。

因此这篇博客就是将图片上传到网上，这样图片就变成了网上的地址，分享给别人也同样能访问。

版本要求

Typora的版本要求：0.9.86以上

点击”帮助=》关于”可以查看现在使用的版本。

这个版本是写这篇博客时的最新版，如果不想麻烦，直接打开软件后提示更新时就顺手更新了吧

安装Picgo

地址 https://github.com/Molunerfinn/PicGo/releases

windows安装.exe文件，mac下载dmg。

修改Typora图片功能设置

点击“文件=》偏好设置=》图像”

修改成如图所示的样子，然后点击下载更新

使用SMMS作为图床

因为需要将图片上传到网上，所以我们需要使用一个可以存储图片的服务器，这里我们使用smms作为图床。

打开网址，注册账号。

https://sm.ms/home/apitoken

注册登录完以后，会自动跳转到一个页面。

不用管这个页面，再次请求之前的网址：https://sm.ms/home/apitoken

复制这里的token，后面会用到。

配置Typora

进入偏好设置=》图像，点击打开配置文件。

修改配置文件，复制下面的代码

{
  "picBed": {
    "uploader": "smms", // 代表当前的默认上传图床为 SM.MS,
    "smms": {
      "token": "这里面的token换成你上个页面的申请的token" //一定要换
    }
  },
  "picgoPlugins": {} // 为插件预留
}

记得换token，保存配置文件，就可以了。

使用阿里云作为图床

之前使用SMMS作为图床，但是太不稳定了，今天一直上传不上去，搞得我很心烦，就决定用阿里云作为图床了，虽然要钱，但是还好不是很贵。

1.首先登陆阿里云，然后点击控制台。

2.然后选择对象存储OSS，并开通服务。

3.点击概况，创建Bucket

4.填写相关信息，注意读写权限一定要选公共读

5.创建完成之后，可以购买一个资源包。

6.购买40G，一年差不多了。

7.获取个人ACCESSkey

8.创建AccessKey

9.打开typora的偏好设置，打开配置文件，修改文件如下：

{
  "picBed": {
    "uploader": "aliyun",
    "aliyun": {
  	"accessKeyId": "LTAI5tMPmnRuSjU3wmEC13TH",   //上述创建的AccessKey的id
 	"accessKeySecret": "60w1Stbt3qSUw6ABilXLeCvSWuFw10",  //上述创建的密码
  	"bucket": "pety", // 存储空间名，即你创建的BUcket的名字
  	"area": "oss-cn-shenzhen", // 存储区域代号
  	"path": "img/", // 自定义存储路径
 	 "customUrl": "", // 自定义域名，注意要加 http://或者 https://
 	 "options": "" // 针对图片的一些后缀处理参数 PicGo 2.2.0+ PicGo-Core 1.4.0+
	}
  },
  "picgoPlugins": {}
}

大功完成！！！！！！！！！
