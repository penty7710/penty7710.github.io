发表于 2021-07-22

在看fastapi的官方文档，然后感觉sql部分挺难的，于是写下一篇博客来记录下。

安装工具

首先要安装两个东西

pip install sqlalchemy

pip install pymysql

前者是一个ORM框架，在ORM中，你创建一个类就会通过SQLAlchemy将其自动转成一张表，在类中的每一个属性就会转成表中的字段。

后者是用来连接数据库的

项目结构


└── sql_app
    ├── __init__.py
    ├── crud.py
    ├── database.py
    ├── main.py
    ├── models.py
    └── schemas.py
  • sql_app是包名，注意是创建一个包，而不是创建一个目录
  • _init_.py是一个空文件，创建sql_app会自动创建这个文件
  • crud.py是用来进行crud操作的文件
  • database.py是用来连接数据库的
  • main.py是用来接受前端请求，类似于controller层
  • models.py是用来创建SQLAlchemy模型的，也就是创建表
  • schemas.py是用来创建Pydentic模型的

database.py

from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, declarative_base

#连接数据库的url
SQLALCHEMY_DATABASES_URL ="mysql+pymysql://root:123456@localhost:3306/python"

#echo=True表示引擎将用repr()函数记录所有语句及其参数列表到日志
engine = create_engine(SQLALCHEMY_DATABASES_URL,encoding="utf8",echo=True)

#SQLAlchemy中，CRUD是通会话进行管理的，所以需要先创建会话
#每一个SessionLocal实例就是一个数据库会话
#commit是指提交事务，将变更保存到数据库
#flush是指发送到数据库语句到数据库，但数据库不一定执行写入磁盘
#autocommit和autoflush是可以不写的
SessionLocal = sessionmaker(autocommit=False,autoflush=False,bind=engine)

#创建基本映射类
Base = declarative_base()

在数据库相关的配置文件中，首先创建一个SQLAlchemy的”engine”，然后创建SessionLocal实例进行会话，最后创建模型类的基类。

models.py

from sqlalchemy import Column, Integer, String, Boolean, ForeignKey
from sqlalchemy.orm import relationship

from .database import Base

#使用database.py创建的Base类创建SQLAlchemy模型
#可以理解为这个是与真正的数据库相关联的类

class User(Base):
    #对应的是数据库中的表名
    __tablename__ = "users"

    #设置表的属性名，使用Column设置，定义了属性的类型
    id = Column(Integer,primary_key=True,index=True)
    email = Column(String(32),unique=True,index=True)
    hashed_password = Column(String(32))
    is_active = Column(Boolean,default=True)

    #设置关系，通过relationship设置，这个不是属性，在数据库中不需要有这一个字段
    #当访问users.items时，会直接关联到Item表，可以直接获取到Item的字段数据
    #后面的back_populates是反向查询，有这个Item表可以通过items.owner访问到User表的字段数据
    #使用了back_populates,那么被关联的数据库也要显示的使用relationship
    #这个其实我也还不是太懂
    items = relationship("Item",back_populates="owner")


class Item(Base):
    __tablename__ = "items"

    id=Column(Integer,primary_key=True,index=True)
    title = Column(String(32),index=True)
    description = Column(String(32) , index=True)
    #设置了外键，使用ForeignKey参数设置，里面填写关联的表和具体的属性
    #注意这里是表名而不是上面的类名
    owner_id  = Column(Integer,ForeignKey("users.id"))

    owner = relationship("User",back_populates="items")

通过数据库配置文件中的基类来创建模型类。

schemas.py

from typing import Optional, List

from pydantic import BaseModel


class ItemBase(BaseModel):
    title:str
    description:Optional[str]=None


#这个是用来创建Item的类，可以理解为前端传过来的要新增的数据
class ItemCreate(ItemBase):
    pass


#继承自ItemBase
#Pydantic风格是声明类型，所以用的是:
#SQLAlchemy风格是定义属性，所以用的是=
class Item(ItemBase):
    id: int
    owner_id: int

    class Config:
        orm_mode=True


class UserBase(BaseModel):
    email:str


class UserCreate(UserBase):
    password:str

class User(UserBase):
    id: int
    is_active: bool
    items: List[Item]=[]

    #Config类用来给Pydantic提供配置信息
    class Config:
        #orm_mode出了可以让Pydantic读取字典类型的数据，还支持Pydantic读取属性数据，比如SQLAlchemy模型的数据
        #这样我们就可以在路径操作函数中直接返回SQLAlchemy数据模型，没有这个配置项是不支持的
        orm_mode = True

定义请求参数模型验证与响应模型验证的Pydantic模型，其中响应模型中设置orm_mode=True参数，表示与ORM模型兼容，因为后续中返回的数据库查询是orm模型，通过设置这个参数可以将orm模型通过pydantic模型进行验证。

crud.py

from sqlalchemy.orm import Session
from . import models, schemas


#通过id查询用户
def get_user(db:Session,user_id: int):
    #返回的是models.User(其实就是users表)中的全部属性，并且条件是user_id = users中的id
    return db.query(models.User).filter(models.User.id==user_id).first()

#通过email查询用户
def get_user_by_email(db:Session,email:str):
    #条件是邮箱相等
    return  db.query(models.User).filter(models.User.email == email).first()

#查询多个用户
def get_users(db:Session,skip:int = 0,limit:int =100):
    #offset表示偏移量，0表示不偏移,1表示偏移1条数据，即从第二条开始
    #limit表示表示限定数量
    #从第一条开始,查询100条数据，返回的是个列表
    return db.query(models.User).offset(skip).limit(limit).all()

#增加数据
#将前端传过来的数据保存到数据库
#传过来的数据是Pydantic模型，需要先转换为SQLAlchemy模型
def create_user(db:Session,user:schemas.UserCreate):
    #密码哈希化(伪造)
    fake_hashed_password = user.password+"pty123"
    #user是Pydantic模型的，重新创建db_user,db_user是SQLALchemy类型的
    db_user = models.User(email=user.email,hashed_password=fake_hashed_password)
    #添加到数据库
    db.add(db_user)
    #提交
    db.commit()
    #刷新
    db.refresh(db_user)
    return db_user

#查询多个项目
def get_items(db:Session,skip:int = 0,limit :int=100):
    return db.query(models.Item).offset(skip).limit(limit).all()


#增加数据
def create_user_item(db:Session,item:schemas.ItemCreate,user_id:int):
    db_item = models.Item(**item.dict(),owner_id = user_id)
    db.add(db_item)
    db.commit()
    db.refresh(db_item)
    return db_item

通过传入数据库连接以及参数等进行数据库操作，包括创建用户、查询用户等，返回的是orm模型对象。

main.py

from typing import List

from fastapi import FastAPI, Depends,HTTPException
from sqlalchemy.orm import Session

from . import schemas, crud, models
from .database import SessionLocal, engine

#创建数据库，就算你没有自己创建表，orm会根据models中的类自动创建表
models.Base.metadata.create_all(bind=engine)

app=FastAPI()

#Dependency
#每一个请求完毕后都需要关闭当前的数据库会话，不同请求使用不同的数据库会话
def get_db():
    db=SessionLocal()
    try:
        yield db
    finally:
        db.close()

@app.post("/users/",response_model=schemas.User)
def create_user(user:schemas.UserCreate,db:Session=Depends(get_db)):
    return crud.create_user(db,user)

@app.get("/user/{user_id}",response_model=schemas.User)
def read_user(user_id: int,db: Session = Depends(get_db)):
    db_user =crud.get_user(db,user_id)
    if db_user is None:
        raise  HTTPException(status_code=404,detail="User not found")
    return db_user

#这里返回多个对象，需要使用List，一开始没用，报错了
@app.get("/users/",response_model=List[schemas.User])
def read_usres(skip: int=0,limit :int=100,db:Session = Depends(get_db)):
  return  crud.get_users(db,skip,limit)

主文件进行数据库初始化、FastAPI实例创建以及处理各种请求。

SQLAlchemy的查询语句

查询执行器

方法 说明
all() 返回一个元素为查询结果的列表
count() 返回查询结果的数量
first() 返回第一个结果或None
first_or_404() 返回第一个结果或404
get(主键) 返回主键对应的对象或None
get_or_404(主键) 返回主键对应的对象或404
paginate(页码，每页条数) 返回一个分页查询结果

查询过滤器

过滤器 说明
filter_by(字段名=值) 等值查询
filter(函数引用/比较运算) 函数或比较查询
limit(限定条数) 限定返回结果数量
offset(偏移条数) 偏移1即从第二条开始查询
order_by(排序字段) 对查询结果根据字段进行排序
options() 针对原查询限定查询的字段
