注解和反射

发表于 2021-04-12 分类于 java

注解

元注解

元注解的作用就是负责注解其他注解，java定义了4个标准的meta-annotation类型，他们被用来提供对其他annotation类型做说明。

  • @Target：用于描述注解的使用范围（即：被描述的注解可以用在什么地方）
  • @Retention：表示需要在什么级别保存该注解信息，用户描述注解的生命周期（source<class<runtime)
  • @Document：说明该注解将包含在javadoc中
  • @Inherited：说明子类可以继承父类中的该注解

自定义注解

使用@Interface自定义注解

import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;i

public class Test {
    //因为age已经有了默认值，可以不用再赋值，如果只有一个参数，还可以直接省略前面你的参数名，直接写参数
    @MyAnnotation(name="pty")
    public void test1(){
        
    }
}


//表示这个注解我们可以用在哪些地方，type表示类，method表示方法，field表示属性
@Target(ElementType.METHOD)
//表示这个注解在什么地方还有效
@Retention(RetentionPolicy.RUNTIME)
@interface MyAnnotation{
    //注解的参数： 参数类型+参数名();
    String name();
    int age() default  12;  //还可以使用default "xxx" 设置默认值
}

反射

概述

反射是java被视为动态语言的关键，反射机制就是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性，这种动态获取的信息以及动态调用对象的方法的功能称为java语言的反射机制。

实际上，我们创建的每一个类都是对象。在加载完类之后，在堆内存的方法区中就产生了一个Class类型的对象，这个类对象包含了完成的类的结构信息。我们可以通过这个对象看到类的结构。

获得反射对象

package com.mp.demo.reflection;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

public class Test {
    public static void main(String[] args) throws ClassNotFoundException {
        //通过反射过去类的Class对象， 使用.forname()方法，参数为实体类的包名+类名
        Class c1 = Class.forName("com.mp.demo.reflection.User");
        System.out.println(c1);
    }

}

@Data
@NoArgsConstructor
@AllArgsConstructor
class User{
    private int id;
    private String name;
    private int age;
}

  • 知道一个类的全类名（包名+类名）,且该类在类路径下，可以通过Class类的静态方法forName()获取，但是可能会抛出异常。

package com.mp.demo.reflection;


public class Test {
    public static void main(String[] args) throws ClassNotFoundException {
        //通过.class 方法获取实例
        Class cl = User.class;
        System.out.println(cl);

        //通过具体对象调用getClass方法创建class实例
        User user = new User();
        Class c2 = user.getClass();
        System.out.println(c2);

        //使用全类名创建class对象
        Class c3 = Class.forName("com.mp.demo.reflection.User");
        System.out.println(c3);

        //获取父类的class对象
        Class c4 = c2.getSuperclass();
        System.out.println(c4);
    }

}

类加载器（JVM 目前先了解）

作用：将class文件字节码内容加载到内存中，并将这些景泰数据转换成为方法区的运行时的数据结构，然后在堆中生成一个代表这个类的java.lang.Class对象，作为方法区中类数据的访问入口。

public class Test {
    public static void main(String[] args) throws ClassNotFoundException {
        //获取系统类的加载器
        ClassLoader systemClassLoader = ClassLoader.getSystemClassLoader();
        System.out.println(systemClassLoader);

        //获取系统类加载器得到父类加载器->拓展类加载器
        ClassLoader parent = systemClassLoader.getParent();
        System.out.println(parent);

        //获取拓展类加载器的父类加载器->根加载器（c/c++)   根（引导类）加载器无法读取到
        ClassLoader parent1 = parent.getParent();
        System.out.println(parent1);


        //测试当前类是哪个加载器加载的
        ClassLoader classLoader = User.class.getClassLoader();
        System.out.println(classLoader);
    }

}

获取运行时类的完整结构（方法，属性，类名，构造器….)

==获取属性和方法、构造器的方法有两种，一种是没加Declared的，一种是加了Declared的。前者只能获取到公共的属性、方法和构造器（即由public修饰的），后者可以获取到所有的，包括被private、protected修饰的==

import java.lang.reflect.Constructor;
import java.lang.reflect.Field;
import java.lang.reflect.Method;

public class Test {
    public static void main(String[] args) throws ClassNotFoundException, NoSuchFieldException, NoSuchMethodException {
        //获取user类对象
        Class c1 = User.class;

        System.out.println("=================名字========================");

        //获取类的名字
        String name = c1.getName();  //获取类的包名+类名
        String name1 = c1.getSimpleName();  //获取类的类名
        System.out.println(name);
        System.out.println(name1);

        System.out.println("=================属性========================");

        //获得类的属性
        Field[] fields = c1.getDeclaredFields();  //获得所有的属性
        for (Field field : fields) {
            System.out.println(field);
        }

        Field name2 = c1.getDeclaredField("name");   //获取指定的属性
        System.out.println(name2);

        System.out.println("=================方法========================");

        //获取类的方法
        Method[] declaredMethods = c1.getDeclaredMethods();  //获取所有的方法
        for (Method declaredMethod : declaredMethods) {
            System.out.println(declaredMethod);
        }
        //获取指定的方法,参数1是方法名，参数二是参数的类型，没有参数填写null
        //为什么要填写参数类型，因为java存在重载
        Method setName = c1.getDeclaredMethod("setName", String.class);
        System.out.println(setName);

        System.out.println("=================构造器========================");

        Constructor[] declaredConstructors = c1.getDeclaredConstructors();  //获取所有的构造器
        for (Constructor declaredConstructor : declaredConstructors) {
            System.out.println(declaredConstructor);
        }
		//此处的参数的类型必须和构造器的参数类型一样。由于我使用了@AllArgsConstructor注解，因此含所有参数的构造器的参数类型的顺序与你定义属性时的顺序一样
        Constructor declaredConstructor = c1.getDeclaredConstructor(int.class, String.class, int.class);
        System.out.println(declaredConstructor);
    }
}

动态创建对象、调用方法、操作属性

import java.lang.reflect.Constructor;
import java.lang.reflect.Field;
import java.lang.reflect.InvocationTargetException;
import java.lang.reflect.Method;

public class Test {
    public static void main(String[] args) throws IllegalAccessException, InstantiationException, InvocationTargetException, NoSuchMethodException, NoSuchFieldException {
        Class c1 = User.class;

        //直接使用newInstance()方法，调用的是无参构造器，如果没有无参构造器，会报错
        Object user = c1.newInstance();
        System.out.println(user);

        System.out.println("=================构造器======================");
        //通过构造器创建实例
        //通过Class对象获取构造器
        Constructor constructor = c1.getDeclaredConstructor(int.class, String.class, int.class);
        User user1 = (User) constructor.newInstance(1, "pty", 12); //通过构造器创建实例，参数为构造器所需的具体的值
        System.out.println(user1);

        System.out.println("====================调用普通方法=================");

        //获取普通方法
        Method setName = c1.getDeclaredMethod("setName", String.class);
        setName.invoke(user1, "彭天怡");   //通过invoke()方法激活方法，第一个参数为执行的对象，第二个是方法需要的具体参数
        System.out.println(user1);

        System.out.println("===================操作属性=====================");

        //获取属性
        Field name = c1.getDeclaredField("name");
        //因为name设置为私有属性，因此不能被外界访问到，使用setAccessible(true)可以使其被访问
        name.setAccessible(true);
        name.set(user1,"pty1"); //设置属性
        name.get(user1);
        System.out.println(user1);
    }
}

总结

  • 如果实体类没有无参构造，不能直接使用newInstance()方法，会报错，应该使用构造器来创建对象
  • 使用构造器.newInstance()创建对象时，参数为构造器具体需要的值
  • 获取到方法后，通过方法.invoke()来激活方法，第一个参数为执行方法的对象，第二个是方法所需的参数。
  • 由于私有方法和私有属性不能被直接操作，因此需要在操作前使用setAccessibel(true)方法，这样就可以操作private修饰的方法和属性了。（setAccessibel(true) 也称为暴力忽略）
  • getMathod()和getField()不仅可以获取到本类的方法和属性，还可以获取到从父类继承的方法和属性。

反射操作泛型

import java.lang.reflect.Method;
import java.lang.reflect.ParameterizedType;
import java.lang.reflect.Type;
import java.util.List;
import java.util.Map;

public class Test {

    public  void test1(Map<String,User> map, List<User> list){
        System.out.println("test1");
    }

    public Map<String,User> test2(){
        System.out.println("test2");
        return null;
    }
    public static void main(String[] args) throws NoSuchMethodException {
        //获取Class对象
        Class c1 = Test.class;
        //获取具体的方法
        Method test1 = c1.getDeclaredMethod("test1", Map.class, List.class);
        //getGenericParameterTypes()返回一个Type对象数组，获取方法的形参类型，如果参数是泛型，将会返回完整的泛型信息
        Type[] genericParameterTypes = test1.getGenericParameterTypes();
        for (Type genericParameterType : genericParameterTypes) {
            System.out.println(genericParameterType);
            //判断参数类型是否属于参数化类型（只要是泛型，且不是数组和本类上定义的泛型）
            if(genericParameterType instanceof ParameterizedType){
                //将genericParameterTyp强转为ParameterizedType类型，并且调用getActualTypeArguments()方法获取参数化类型中的实际参数
                Type[] actualTypeArguments = ((ParameterizedType) genericParameterType).getActualTypeArguments();
                for (Type actualTypeArgument : actualTypeArguments) {
                    System.out.println(actualTypeArgument);
                }
            }

        }
        System.out.println("=============方法返回值的泛型============");
        //操作与获取上面方法的参数类型一致，不同的是获取的是返回值的参数类型 getGenericReturnType()
        Method test2 = c1.getDeclaredMethod("test2", null);
        Type genericReturnType = test2.getGenericReturnType();
        System.out.println(genericReturnType);
        if(genericReturnType instanceof  ParameterizedType){
            Type[] actualTypeArguments = ((ParameterizedType) genericReturnType).getActualTypeArguments();
            for (Type actualTypeArgument : actualTypeArguments) {
                System.out.println(actualTypeArgument);
            }
        }
    }
}

反射操作注解

User类 + 自定义注解

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

@Data
@NoArgsConstructor
@AllArgsConstructor
@Table("用户表")
public class User{
    @FieldUser(columnName = "id",type = "int")
    private int id;
    @FieldUser(columnName = "name",type = "String")
    private String name;
    @FieldUser(columnName = "age",type = "int")
    private int age;
}

//自定义注解
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@interface Table{
    String value();
}

@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
@interface FieldUser{
    String columnName();
    String type();
}

测试类

import java.lang.annotation.Annotation;
import java.lang.reflect.Field;

public class Test {
    public static void main(String[] args) throws NoSuchFieldException {
        Class c1 = User.class;
        //获取类全部的注解
        Annotation[] annotations = c1.getAnnotations();
        for (Annotation annotation : annotations) {
            System.out.println(annotation);
        }

        //获取类特定的注解,并强转为特定的类型
        Table annotation = (Table) c1.getAnnotation(Table.class);
        String value = annotation.value();
        System.out.println(value);

        //获取属性特定的注解
        //先获取属性
        Field name = c1.getDeclaredField("name");
        //获取属性的注解，这里不需要进行强转，已经是FieldUser类了
        FieldUser annotation1 = name.getAnnotation(FieldUser.class);
        //获取注解的值
        System.out.println(annotation1.columnName());
        System.out.println(annotation1.type());
    }
}

==只有当注解的@Retention=RUNTIME时，才能在反射中被获取到==
